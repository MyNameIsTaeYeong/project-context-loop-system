# Phase 9.8 — code_doc → 기존 파이프라인 연결

- **일시**: 2026-04-14
- **범위**: Phase 9.8 — LLM 생성 문서를 기존 처리 파이프라인(chunker → embedder → graph_extractor)에 연결
- **브랜치**: `claude/prepare-phase-9.8-NbTdP`

## 작업 내용

### 1. CoordinatorAgent 파이프라인 의존성 추가

**문제**: `run_and_store()`가 code_file_summary/code_summary/code_doc를 DB에 저장하지만, 기존 파이프라인(LLM 분류 → 청킹 → 임베딩 → 그래프 추출)을 거치지 않아 벡터 검색(ChromaDB)이나 그래프 탐색에서 검색 불가.

**구현:**
- `CoordinatorAgent.__init__`에 keyword-only 파이프라인 의존성 4개 추가:
  - `vector_store: VectorStore | None = None`
  - `graph_store: GraphStore | None = None`
  - `pipeline_llm_client: LLMClient | None = None`
  - `embedding_client: Embeddings | None = None`
- `TYPE_CHECKING` 블록으로 타입만 임포트 (런타임 의존성 최소화)

### 2. store 메서드 반환값 변경

- `store_directory_summary()` → `tuple[int, bool]` (doc_id, needs_pipeline)
- `store_file_summary()` → `tuple[int, bool]`
- `store_category_document()` → `tuple[int, bool]`
- 신규/변경 시 `True`, 무변경(동일 content_hash) 시 `False`

### 3. _process_through_pipeline() 헬퍼

- `_pipeline_available` 프로퍼티: 4개 의존성 모두 설정 확인
- `_process_through_pipeline(document_id)`:
  - 의존성 미설정 시 건너뜀 (None 반환)
  - `process_document()` 지연 임포트 + 호출
  - `PipelineConfig`는 앱 설정(config.yaml)에서 읽음
  - 실패 시 예외 전파 안 함 (try/except → 로그 + None 반환)

### 4. run_and_store() 파이프라인 호출 연결

- Level 1 (code_file_summary): 저장 + document_sources → 파이프라인
- Level 2 (code_summary): 저장 + document_sources → 파이프라인
- Level 3 (code_doc): 저장 + document_sources → 파이프라인
- git_code: 원본 코드이므로 파이프라인 **미적용**
- 파이프라인 처리 집계 로깅 (성공/실패 카운트)

### 5. git_sync.py 파이프라인 의존성 전달

- `start_sync()`: 기존 config/meta_store 외에 vector_store/graph_store/llm_client/embedding_client를 FastAPI Depends로 주입
- `_run_sync()`: 위 의존성을 keyword-only 파라미터로 수신 → CoordinatorAgent에 전달
