### 1. CLAUDE.md 대시보드 중심으로 기획 변경

- 문서 입력 방식 3가지 정의: Confluence API 임포트, 파일 업로드(.md 등), 직접 마크다운 작성
- LLM 기반 저장 방식 판단 추가: 문서 내용을 LLM이 분석하여 텍스트 청크(벡터DB) / 그래프DB / 혼합 중 자동 결정
- 대시보드 화면 구성 정의: 메인 대시보드, 문서 상세 뷰(원본/청크/그래프/메타데이터 탭), Confluence 임포트 화면, 마크다운 에디터
- 프로젝트 구조 변경: `connectors/` → `ingestion/`, `classifier.py`, `graph_extractor.py`, `graph_store.py` 등 신규 모듈
- DB 스키마 변경: `graph_nodes`, `graph_edges` 테이블 추가, `documents.storage_method` 컬럼 추가

### 2. MCP Server 계층 추가
- 사내 LLM 앱이 MCP 프로토콜(stdio/SSE)로 질의하면 저장된 지식에서 컨텍스트를 검색·조립하여 응답
- 4개 MCP Tools 정의: `search_context`, `list_documents`, `get_document`, `get_graph_context`
- 컨텍스트 조립 플로우 설계: 질의 임베딩 → 벡터 유사도 검색 + 그래프 탐색 → 결과 병합/정렬 → 응답
- `mcp/` 모듈 추가 (`server.py`, `tools.py`, `context_assembler.py`)
- 기술 스택에 `mcp` Python SDK + FastMCP 추가
- 설정 파일에 `mcp` 섹션 추가
- 구현 순서에 Phase 5 (MCP Server) 신설, 기존 Phase가 6~7로 밀림

### 3. 문서 변경 감지 및 재처리 설계
- 변경 감지: `content_hash` 비교 (Confluence는 추가로 `version` 비교)
- 재처리 전략: **Delete & Recreate** — 기존 파생 데이터 전부 삭제 후 재생성
- 그래프 엔티티 관리: 문서 소유권 기반 삭제, 엔티티 병합 테이블, 고아 엣지 정리
- 재처리 이력: `processing_history` 테이블에 이전/이후 storage_method 기록
- Confluence 자동 동기화: 주기적 증분 동기화 + 대시보드 수동 트리거
- DB 스키마 보강: `documents.version`, `documents.status`에 "changed" 추가, `processing_history`에 `prev_storage_method`/`new_storage_method` 분리
