## 작업 내역

### 1. HTML→마크다운 변환: LLM → markdownify 라이브러리 전환

**파일**: `src/context_loop/ingestion/mcp_confluence.py`

- **변경 전**: `convert_html_to_markdown()`이 `LLMClient`를 사용하여 HTML을 마크다운으로 변환 (비동기, LLM 호출)
- **변경 후**: `markdownify` 라이브러리를 사용한 동기 함수로 전환
- **이유**: LLM 호출 시 timeout/비용 문제 해결, 결정론적 변환 보장
- **영향 범위**:
  - `mcp_confluence.py`: `LLMClient` import 제거, `_HTML_TO_MD_SYSTEM` 프롬프트 삭제, 함수 시그니처 변경
  - `web/api/confluence_mcp.py`: `import_pages` 엔드포인트에서 `llm_client` 의존성 제거
  - `tests/test_ingestion/test_mcp_confluence.py`: LLM mock 기반 테스트 → 실제 라이브러리 변환 결과 검증으로 변경
- **추가 의존성**: `markdownify>=1.2.2` (`beautifulsoup4`, `soupsieve` 자동 설치)

### 2. classify_document LLM max_tokens 증가 (256 → 1024)

**파일**: `src/context_loop/processor/classifier.py`

- **변경 전**: `max_tokens=256`으로 LLM 응답 토큰 제한
- **변경 후**: `max_tokens=1024`로 증가
- **이유**: 문서 분류 시 LLM 응답이 잘려 JSON 파싱 에러 발생
- **영향 범위**: `classify_document()` 함수의 LLM 호출 부분만 변경
