# Phase 9.6 — Category Agent 구현

- **일시**: 2026-04-08
- **범위**: Phase 9.6 — Level 3 상품×카테고리별 관점 문서 생성

## 작업 내용

### 1. `LLMCategoryAgent` 구현 (`ingestion/category_agent.py`)

Worker Agent가 생성한 관점 중립적 Level 2 디렉토리 문서를 종합하여,
카테고리별 관점 문서(Level 3)를 생성하는 에이전트.

**핵심 설계:**
- `CategoryAgentProtocol` (coordinator.py:110-119) 구현
- config에 정의된 카테고리 프롬프트를 system 프롬프트로 사용 (D-028)
- orchestrator 엔드포인트(Opus급 고성능 모델) 사용 (D-029)
- 글자수 기반 동적 배치 + Map-Reduce로 대형 입력 타임아웃 방지

**Map-Reduce 처리 방식:**
- 디렉토리 요약을 `max_chars_per_batch` (기본 15,000자) 기준으로 동적 분할
- 배치 1개: 단일 LLM 호출 (기존과 동일)
- 배치 2개+: Map(배치별 부분 관점 문서 병렬 생성) → Reduce(부분 문서 종합)
- Map: max_tokens=8192, Reduce/단일: max_tokens=16384
- Map 배치 일부 실패 허용. 부분 문서 1개만 남으면 Reduce 생략
- `graph_extractor.py`의 map-reduce 패턴을 참조

**프롬프트 3종:**
- 단일 호출 (`_CATEGORY_USER_TEMPLATE`): 모든 디렉토리 한번에
- Map (`_MAP_USER_TEMPLATE`): 배치별 부분 문서 생성
- Reduce (`_REDUCE_USER_TEMPLATE`): 부분 문서 종합

### 2. `EndpointLLMClient` 타임아웃 지원 (`processor/llm_client.py`)

- `timeout` 파라미터 추가 (기본 600초, connect 10초)
- `httpx.Timeout`으로 세밀한 제어
- `git_config.build_llm_client(agent, timeout=...)` 으로 에이전트별 설정 가능

### 3. 테스트 (`tests/test_ingestion/test_category_agent.py`)

24개 테스트 작성, 전체 통과:
- 기본 동작 (4개): 정상 반환, product/category 필드, source_directories, 공백 제거
- 프롬프트 (4개): system 프롬프트, user 프롬프트, target_audience, LLM 파라미터
- 엣지 케이스 (3개): 빈 입력, LLM 오류 전파, enable_thinking
- 배치 분할 (5개): 단일 배치, map-reduce 전환, 경계값, 동적 분할, 초대형 디렉토리
- Map-Reduce (8개): 병렬 호출, 부분 실패 허용, 전체 실패, Map/Reduce max_tokens,
  Reduce 프롬프트 내용, 단일 생존 배치 Reduce 생략, source_directories 완전성

### 4. 수동 테스트 스크립트 (`scripts/run_category_agent.py`)

- `--input-dir` 모드: Worker 출력의 `_level2_summary.md` 파일을 로드하여 Category Agent 실행
- `--full-pipeline` 모드: Git clone → Worker → Category Agent 순차 실행
- `--categories` 옵션: 특정 카테고리만 선택 실행 가능
- 결과를 `scripts/output/{product}/category/{category}.md`에 저장

## Coordinator 통합

Coordinator는 이미 Category Agent 호출 로직이 구현되어 있어 수정 불필요:
```python
# coordinator.py:315-329
if self._category_agent and result.directory_summaries:
    categories = self._git_config.get_category_list()
    cat_tasks = [self._run_category_agent(product, cat, result.directory_summaries) ...]
```

`LLMCategoryAgent` 인스턴스를 `category_agent` 인자로 전달하면 바로 동작.
