# 2026-03-11 — Phase 2 문서 입력 파이프라인 구현

## 수행 내용

### 1. 문서 파서 (공통 유틸리티)
- `src/context_loop/processor/parser.py` 구현
  - `is_supported_file()`: 지원 파일 형식 확인 (.md, .txt, .html)
  - `compute_content_hash()`: SHA-256 콘텐츠 해시
  - `html_to_markdown()`: 기본 HTML→마크다운 변환 (헤딩, 볼드, 이탤릭, 링크, 리스트, 엔티티)
  - `normalize_content()`: 포맷별 정규화
  - `extract_title_from_content()`: 마크다운에서 제목 자동 추출

### 2. 파일 업로드 처리 (2.1)
- `src/context_loop/ingestion/uploader.py` — FileUploader 클래스
  - `upload_file()`: 파일 경로에서 읽어 문서 등록
  - `upload_content()`: 문자열 콘텐츠 직접 등록 (웹 업로드용)
  - 지원 형식 검증, 자동 제목 추출, HTML 자동 변환

### 3. 마크다운 직접 작성 (2.2)
- `src/context_loop/ingestion/editor.py` — ManualEditor 클래스
  - `create_document()`: 새 마크다운 문서 생성
  - `update_document()`: 기존 문서 수정 (content_hash 비교로 변경 감지)

### 4. Confluence API 임포트 (2.3)
- `src/context_loop/ingestion/confluence.py`
  - `ConfluenceClient`: REST API 클라이언트 (Cloud/DataCenter 인증, 스페이스/페이지 조회)
  - `ConfluenceImporter`: 페이지/스페이스 단위 임포트, 중복 감지

### 5. Confluence 증분 동기화 (2.4)
- `src/context_loop/sync/engine.py` — SyncEngine 클래스
  - `sync_once()`: 단일 동기화 실행 (로컬 문서 vs 원격 해시 비교)
  - `start()`/`stop()`: 백그라운드 주기적 동기화

### 6. 문서 변경 감지 및 재처리 (2.5)
- `src/context_loop/processor/pipeline.py` — ProcessingPipeline 클래스
  - `process_document()`: 문서 처리 (Phase 3 연동 준비)
  - `reprocess_document()`: Delete & Recreate 전략
  - `check_and_reprocess()`: 콘텐츠 변경 확인 후 조건부 재처리
