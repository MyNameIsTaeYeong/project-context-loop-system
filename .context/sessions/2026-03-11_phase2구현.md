# 2026-03-11 — Phase 2 문서 입력 파이프라인 구현

## 수행 내용

### 1. 파일 업로드 처리 (2.1)
- `src/context_loop/ingestion/uploader.py` 구현
  - `compute_content_hash()`: SHA-256 기반 변경 감지용 해시
  - `read_file()`: .md/.txt/.html 지원, 미지원 확장자는 `UnsupportedFileTypeError`
  - `upload_file()`: 신규 생성/변경없음/변경감지 3가지 케이스 처리, processing_history 기록

### 2. 마크다운 직접 작성 저장 (2.2)
- `src/context_loop/ingestion/editor.py` 구현
  - `save_document()`: document_id 없으면 신규 생성, 있으면 hash 비교 후 갱신
  - 신규/변경없음/변경됨 케이스 처리

### 3. Confluence API 임포트 (2.3)
- `src/context_loop/ingestion/confluence.py` 구현
  - `_html_to_markdown()`: 정규식 기반 경량 HTML→MD 변환
  - `ConfluenceClient`: Cloud(Basic Auth) / Data Center(Bearer Token) 지원
    - `list_spaces()`, `list_pages()`, `get_page()`, `get_page_content_as_markdown()`
    - 페이지네이션 cursor 기반 자동 처리
  - `import_page()`: 단일 페이지 임포트 (신규/변경없음/변경감지)
  - `import_space()`: 스페이스 전체 임포트

### 4. Confluence 증분 동기화 (2.4)
- `src/context_loop/sync/engine.py` 구현
  - `SyncResult`: 생성/갱신/변경없음/오류 집계 데이터 클래스
  - `sync_space()`: since 파라미터로 증분 동기화 지원
  - `SyncEngine`: 백그라운드 주기적 동기화 관리
    - `start()`, `stop()`, `sync_now()` 제공

### 5. 문서 변경 감지 및 재처리 파이프라인 (2.5)
- `src/context_loop/processor/reprocessor.py` 구현
  - `check_and_mark_changed()`: content_hash 비교 후 변경 시 status='changed'로 마킹
  - `delete_derived_data()`: 청크 + 그래프 노드/엣지 삭제 (재처리 1단계)
  - `start_reprocessing()`: 파생 데이터 삭제 + status='processing' 전환
  - `complete_reprocessing()`: status='completed'/'failed' + 처리 이력 완료
  - `get_pending_documents()`: pending/changed 문서 목록 조회

  페이지 메타에 항상 포함되지 않을 수 있어 fallback 처리 포함
