# 세션: Phase 1 기반 구조 구현

- **일시**: 2026-03-01
- **목표**: Phase 1 (기반 구조) 전체 구현

## 수행 내용

### 1.1 프로젝트 스캐폴딩
- `pyproject.toml` 생성 (hatchling 빌드, 의존성 정의, ruff/mypy/pytest 설정)
- `src/context_sync/` 패키지 구조 생성 (connectors, processor, storage, sync, query, web)
- `tests/` 디렉토리 구조 생성 (test_connectors, test_processor, test_storage, test_sync)
- `config/default.yaml` 기본 설정 템플릿 생성
- `cli.py` CLI 스켈레톤 구현 (init, start, stop, status, ask, sync, config, autostart)

### 1.2 설정 관리 모듈
- `config.py` 구현:
  - `_deep_merge()`: 딕셔너리 재귀 병합
  - `load_default_config()`: 번들 기본값 로드
  - `load_config()`: 사용자 설정 + 기본값 병합
  - `save_config()`: YAML 파일 저장
  - `get_data_dir()`: 데이터 디렉토리 경로 해석
  - `init_config()`: 초기 설정 파일 생성

### 1.3 인증 모듈
- `auth.py` 구현:
  - `store_token()`: 키체인에 토큰 저장
  - `get_token()`: 키체인에서 토큰 조회
  - `delete_token()`: 키체인에서 토큰 삭제
  - `has_token()`: 토큰 존재 여부 확인
  - 서비스명 `context-sync`, 키 형식 `{source_type}:{key_name}`

### 테스트
- `test_config.py`: 12개 테스트 (deep merge, 로드, 저장, 초기화)
- `test_auth.py`: 8개 테스트 (keyring 모킹)
- `test_cli.py`: 5개 테스트 (CLI 명령어)
- **총 26개 테스트 모두 통과**
