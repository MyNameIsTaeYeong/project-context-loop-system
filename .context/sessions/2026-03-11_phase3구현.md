# 2026-03-11 — Phase 3 LLM 저장 방식 판단 + 처리 구현

## 수행 내용

### 1. LLM 클라이언트 추상화 (3.1 선행)
- `src/context_loop/processor/llm_client.py` 구현
  - `LLMClient` 추상 기본 클래스
  - `AnthropicClient`: claude-haiku-4-5 기본, AsyncAnthropic 사용
  - `OpenAIClient`: gpt-4o-mini 기본, AsyncOpenAI 사용
  - `extract_json()`: 마크다운 코드 블록 및 순수 JSON 모두 파싱

### 2. LLM Classifier (3.1)
- `src/context_loop/processor/classifier.py` 구현
  - 문서를 chunk/graph/hybrid로 분류하는 프롬프트 설계
  - 파싱 실패 또는 알 수 없는 분류 시 "chunk"로 안전 폴백
  - `classify_document(client, title, content) → (method, reason)`

### 3. 텍스트 청킹 (3.2)
- `src/context_loop/processor/chunker.py` 구현
  - tiktoken 기반 토큰 카운팅 (네트워크 접근 실패 시 문자 기반 폴백)
  - 단락(\\n\\n) 경계 우선 분할, 초과 시 강제 분할
  - chunk_overlap으로 인접 청크 간 겹침 처리
  - `chunk_text(text, chunk_size, chunk_overlap) → list[Chunk]`

### 4. 임베딩 클라이언트 (3.3 일부)
- `src/context_loop/processor/embedder.py` 구현
  - `EmbeddingClient` 추상 클래스
  - `OpenAIEmbeddingClient`: 배치 처리 지원 (100개 단위)
  - `LocalEmbeddingClient`: sentence-transformers 기반 (asyncio executor)

### 5. ChromaDB 벡터 저장소 (3.3)
- `src/context_loop/storage/vector_store.py` 구현
  - ChromaDB PersistentClient, cosine 유사도
  - `add_chunks()`, `delete_by_document()`, `search()`, `count()`

### 6. 그래프 엔티티/관계 추출 (3.4)
- `src/context_loop/processor/graph_extractor.py` 구현
  - LLM 기반 엔티티/관계 추출 프롬프트 설계
  - `Entity`, `Relation`, `GraphData` 데이터 클래스
  - 파싱 실패 시 빈 GraphData 반환

### 7. 그래프 저장소 (3.5 + 3.6)
- `src/context_loop/storage/graph_store.py` 구현
  - NetworkX DiGraph 인메모리 인덱스 + SQLite 영속 저장
  - `save_graph_data()`: 엔티티 → 노드, 관계 → 엣지 저장. 소스 엔티티 없는 관계 스킵
  - `delete_document_graph()`: 문서 소유 노드/엣지 삭제 + 고아 엣지 정리
  - `get_neighbors(entity_name, depth)`: NetworkX 기반 BFS 탐색
  - `load_from_db()`: 재시작 시 DB에서 그래프 재구성

### 8. 파이프라인 통합
- `src/context_loop/processor/pipeline.py` 구현
  - `process_document()`: 전체 처리 흐름 조율
    1. start_reprocessing (파생 데이터 삭제)
    2. LLM classify
    3. chunk → 청킹 + 임베딩 + ChromaDB + SQLite chunks
    4. graph → graph_extractor + GraphStore
    5. complete_reprocessing
