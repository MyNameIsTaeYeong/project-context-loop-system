# 2026-04-08 Phase 9.5 Worker Agent 구현 + Coordinator 리팩토링

## 작업 내용

### 1. LLMWorkerAgent 구현 (`ingestion/worker_agent.py`)
- Protocol(`WorkerAgent`)을 만족하는 실제 LLM 호출 구현체
- **Level 1 (파일 요약)**: worker 엔드포인트(경량 모델), max_tokens=4096
- **Level 2 (디렉토리 문서)**: synthesizer 엔드포인트(중간 모델), max_tokens=8192
- 관점 중립적 사실 요약 (관점 부여는 Category Agent가 담당)
- `enable_thinking=False`로 Qwen3 등 추론 모델의 thinking 출력 방지
- `asyncio.Semaphore(max_concurrent_files=5)`로 파일 요약 동시성 제한
- `max_file_tokens`(기본 12000) 초과 시 내용 절삭
- 개별 파일 실패 시 `[요약 실패: ...]`로 대체, 나머지 정상 진행
- 모든 파일 실패 시 synthesizer 호출 없이 에러 메시지 반환

### 2. Coordinator `_process_repository` 리팩토링
- **버그 수정**: `parse_product_scopes()`에 `clone_dir`을 전달하지 않아 자동 탐지가 동작하지 않던 문제 해결
- **D-030 결정**: `sync_repository()` → `clone_or_pull()`로 교체
  - Worker/Category Agent는 `collect_files()`로 디스크에서 직접 읽으므로 git_code DB 저장 불필요
  - git_code 저장 + document_sources 연결은 Phase 9.7에서 함께 구현
- **구조 개선**: 전체 파일 수집 → 상품별 분류 dict 대신, scopes 직접 순회로 단순화

### 3. 수동 테스트 스크립트 (`scripts/run_worker_agent.py`)
- `--local-dir` 모드: Git clone 없이 로컬 파일 직접 분석
- `--full-pipeline` 모드: config의 레포를 clone → 상품별 Worker 실행
- 결과를 `scripts/output/{product}/{directory}/`에 마크다운 파일로 저장
  - `_level1_{filename}.md`: 파일별 요약
  - `_level2_summary.md`: 디렉토리 종합 문서
- `.gitignore`에 `scripts/output/` 추가
