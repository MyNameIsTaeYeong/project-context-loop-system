# 2026-03-16 세션: Phase 5 MCP Server + Phase 6 채팅 인터페이스 구현

## 수행 내용

### Phase 5: MCP Server 구현
1. **MCP 서버 기본 구조** (`mcp/server.py`)
   - FastMCP 기반, stdio/SSE 전송 모두 지원
   - 앱 시작 시 MetadataStore, VectorStore, GraphStore, 임베딩 클라이언트 초기화
   - `run_stdio()`, `run_sse(port)` 진입점 제공

2. **MCP Tool 4종** (`mcp/tools.py`)
   - `search_context`: 질의 → 벡터 유사도 검색 + 그래프 탐색 → 컨텍스트 조립·반환
   - `list_documents`: 문서 목록 조회 (source_type, status 필터)
   - `get_document`: 문서 원본/청크/그래프 데이터 조회 (format 파라미터)
   - `get_graph_context`: 엔티티 중심 그래프 관계 탐색 (depth 지원)

3. **컨텍스트 조립 모듈** (`mcp/context_assembler.py`)
   - 벡터 검색: 질의 임베딩 → ChromaDB 유사도 검색 → 청크 결과 포맷팅
   - 그래프 탐색: 질의 키워드 → 엔티티 매칭 → 관계 탐색
   - 두 결과를 병합하여 하나의 컨텍스트 텍스트로 반환

4. **의존성 추가**: `pyproject.toml`에 `mcp>=1.0.0` 추가

### Phase 6: 채팅 인터페이스 + 출처 표시

1. **채팅 API** (`web/api/chat.py`)
   - `POST /api/chat` — RAG 파이프라인: context_assembler → LLM 호출 → 답변 + 출처 반환
   - `GET /chat` — 채팅 페이지 렌더링
   - 시스템 프롬프트로 사내 지식 기반 답변 유도

2. **채팅 UI** (`templates/chat.html` + `static/js/chat.js`)
   - Alpine.js 기반 대화 인터페이스
   - 메시지 영역 (스크롤, 로딩 표시)
   - 출처 링크 표시 (문서 상세 페이지로 이동, 유사도 백분율)

3. **출처 추적 기능** (`context_assembler.py` 확장)
   - `Source` 데이터클래스: document_id, title, similarity
   - `AssembledContext` 데이터클래스: context_text + sources
   - `assemble_context_with_sources()` 함수: MCP Tool과 채팅 API 양쪽에서 재사용

4. **네비게이션 업데이트**: base.html에 Chat 링크 추가

## 추가 작업: 그래프 탐색 개선 — 임베딩 기반 엔티티 매칭 (C안)

### 문제점
1. 키워드 완전 일치만 지원 ("게이트웨이" → "Gateway" 매칭 불가)
2. 질의 의도와 무관하게 항상 그래프 탐색 실행
3. depth=1 고정

### 개선 내용 (D-014)
1. **GraphStore 엔티티 임베딩 캐시** (`graph_store.py`)
   - `_entity_embeddings`: node_id → (name, embedding) 캐시
   - `build_entity_embeddings()`: 전체 엔티티 이름 임베딩 생성 (증분)
   - `search_entities_by_embedding()`: 코사인 유사도 기반 검색 (threshold, top_k)
   - save/delete 시 캐시 자동 무효화

2. **context_assembler 임베딩 기반 그래프 탐색** (`context_assembler.py`)
   - `_search_graph_by_embedding()`: 질의 임베딩 → 엔티티 매칭 → 이웃 탐색
   - 매칭 엔티티 없으면 탐색 스킵 (None 반환)
   - 매칭 수 기반 depth 동적 결정 (≥3: depth=1, <3: depth=2)
   - 질의 임베딩을 벡터 검색 + 엔티티 매칭에 재사용 (1회 생성)

3. **앱 시작 시 임베딩 사전 빌드** (`app.py`)
   - lifespan에서 `build_entity_embeddings()` 호출
