# 2026-03-11 — Phase 4 웹 대시보드 구현

## 수행 내용

### 기술 결정
- 웹 프레임워크: FastAPI + Jinja2 + HTMX + Alpine.js (I-001 해결)
- 그래프 시각화: vis.js (I-002 해결)
- CSS 프레임워크: Pico CSS (CDN)
- 마크다운 에디터: EasyMDE (CDN)

### 1. 기반 구조 (4.1)
- `src/context_loop/web/app.py`: FastAPI 앱 팩토리 + lifespan (스토어 초기화/정리)
- `src/context_loop/web/dependencies.py`: DI 함수 (get_meta_store, get_templates 등)
- `src/context_loop/web/api/stats.py`: 통계 JSON + HTML 파셜 엔드포인트
- `src/context_loop/web/api/documents.py`: 메인 라우터 (대시보드, 문서 상세, 에디터, CRUD)

### 2. 대시보드 (4.1)
- `templates/dashboard.html`: 통계 카드 + 필터 바 + 문서 목록 (HTMX lazy load)
- `templates/partials/document_stats.html`: 4개 통계 카드 (문서, 청크, 노드, 엣지)
- `templates/partials/document_list.html`: 문서 테이블 (소스/상태/저장방식 배지)

### 3. 문서 상세 뷰 (4.2)
- `templates/document_detail.html`: Alpine.js 탭 + HTMX 파셜 로드
- `templates/partials/tab_original.html`: 원본 마크다운 표시
- `templates/partials/tab_chunks.html`: 청크 목록 (인덱스, 토큰 수, 내용)
- `templates/partials/tab_metadata.html`: 문서 정보 + 처리 이력 타임라인

### 4. 그래프 시각화 (4.3)
- `templates/partials/tab_graph.html`: vis.js 컨테이너 + JSON 데이터 주입
- `static/js/graph.js`: vis.js Network 초기화 (그룹별 컬러, physics 설정)

### 5. 마크다운 에디터 (4.5)
- `templates/editor.html`: 새 문서/기존 문서 에디터 (EasyMDE CDN)
- `static/js/editor.js`: EasyMDE 초기화 (툴바 설정)
- 문서 생성 (`POST /api/documents`) 및 수정 (`PUT /api/documents/{id}`) API

### 6. 파일 업로드 (4.6)
- `src/context_loop/web/api/upload.py`: FastAPI UploadFile 처리 (.md/.txt/.html)
- 대시보드에 파일 업로드 입력 통합

### 7. Confluence 임포트 UI (4.4)
- `src/context_loop/web/api/confluence.py`: 연결 설정, 스페이스/페이지 조회, 임포트 API
- `templates/confluence.html`: 연결 폼 + Alpine.js 스페이스 브라우저 + 페이지 선택 임포트

### 8. 공통
- `templates/base.html`: 레이아웃 (Pico CSS + HTMX + Alpine.js CDN, 네비게이션)
- `static/css/app.css`: 배지, 토스트, 그래프 컨테이너 스타일
- `static/js/app.js`: HTMX 토스트 이벤트 핸들러, 에러 핸들러

### 9. 테스트
- `tests/test_web/conftest.py`: 격리된 스토어 + AsyncClient 픽스처
- `tests/test_web/test_app.py`: 6개 (대시보드, 통계, 목록, 404, 에디터)
- `tests/test_web/test_api_documents.py`: 10개 (CRUD, 탭 파셜, 필터, 상태)
- `tests/test_web/test_api_upload.py`: 3개 (MD, TXT, 미지원 형식)
- 전체 누적 93개 통과 (Phase 1: 15 + Phase 2: 28 + Phase 3: 31 + Phase 4: 19)
