🔄 Context Loop

사내 지식을 LLM 컨텍스트로 변환·저장하고, 웹 대시보드와 MCP 서버로 관리하는 통합 지식 시스템

이 프로젝트는 무엇을 위한 건가요?

회사의 지식은 Confluence, 위키, 코드 저장소, 개인 메모 등 여러 곳에 흩어져 있습니다. Context Loop는 이렇게 분산된 지식을 한곳으로 자동 수집·처리하여, 웹 대시보드에서 시각적으로 관리하고 MCP 프로토콜을 통해 Claude 같은 사내 LLM 애플리케이션에 구조화된 컨텍스트로 제공합니다.

주요 기능

📥

4가지 입력 경로

  • Confluence MCP — 페이지·서브트리·스페이스 단위 임포트
  • 파일 업로드 — md·txt·html
  • 마크다운 작성 — 내장 에디터
  • Git 코드 — 심볼 단위 수집

  • 지식 그래프 시각화
  • 원본/청크/그래프/이력 탭

🔧 Git 코드는 어떻게 임포트되나요?

설정한 Git 저장소의 코드를 심볼 단위로 인덱싱합니다. 저장소는 URL·브랜치·상품(파일 경로 스코프) 단위로 여러 개 설정할 수 있습니다.

1
저장소 동기화

저장소를 clone/pull 하고 커밋 해시를 기록(증분 대비)

2
파일 수집

상품 경로(glob)·제외 패턴 + 지원 확장자·크기 한도로 필터링 (node_modules·build 자동 제외)

3
심볼 추출 (AST)

함수·클래스·메서드를 정적 분석으로 추출. Python=ast, Go·Java·TS·JS 지원. LLM 호출 없음(비용 0)

4
멀티뷰 임베딩

심볼 1개 = 청크 1개. 본문(코드)과 메타(이름·시그니처)를 별도 view로 임베딩

5
import 그래프

파일↔모듈 import, 클래스→메서드 contains 관계 추출. 같은 모듈 import는 한 노드로 병합

6
증분 저장

content_hash 비교로 변경된 파일만 재처리(Delete & Recreate)

🛠️ 인덱싱은 어떻게 이루어지나요?

문서가 들어오면 다음 6단계를 거쳐 검색 가능한 인덱스로 변환합니다.

1
수집

원본과 content_hash를 함께 보관

2
전처리

HTML→마크다운, 구조 정규화 (코드는 AST 분석)

3
청킹

토큰 단위(문서) 또는 심볼 단위(코드)로 분할

4
임베딩

본문·메타·가상질문을 각각 별도 view로 임베딩

5
그래프 추출

엔티티·관계 추출, 표기 변형은 병합

6
저장

ChromaDB·NetworkX+SQLite·SQLite에 저장

원본이 바뀌면 content_hash 비교로 변경을 감지해, 기존 파생 데이터를 지우고 다시 만드는 Delete & Recreate 방식으로 재처리합니다.

💡 가상 질문 생성 (멀티 벡터 인덱싱)

임베딩 단계에서 LLM이 각 섹션마다 "이 섹션이 답할 수 있는 자연스러운 질문"을 미리 만들어 별도 view로 등록합니다(문서당 1회 호출, 섹션당 약 5개). 사용자의 자연어 질의가 본문보다 질문 형태의 임베딩과 더 가깝게 매칭되어 검색 재현율·정밀도가 향상됩니다. 생성된 질문은 문서 상세의 청크 탭에서 볼 수 있습니다.

🔍 검색은 어떻게 이루어지나요?

벡터 검색과 지식 그래프 탐색을 함께 수행해 하나의 컨텍스트로 조립합니다.

1
질의 임베딩

질의를 벡터로 변환 (선택: HyDE로 가상 답변 보강)

2
벡터 검색

ChromaDB 코사인 유사도 → 문서 단위 dedup → top-k

3
그래프 탐색

LLM 플래너가 엔티티 식별 → 깊이 1 양방향 BFS

4
리랭킹(선택)

cross-encoder로 재점수화해 정밀도 향상

5
병합·정렬

벡터+그래프 결과 통합, 중복 제거 후 유사도 정렬

검색 결과로 무엇을 돌려주나요?

📄 ① 포맷된 컨텍스트 텍스트

## 관련 문서, ## 관련 그래프 컨텍스트, ## 그래프 연결 문서 등 섹션으로 구성된 문자열. 항목마다 제목·유사도·섹션 경로(가상 질문 매칭 시 해당 질문)가 붙어 LLM에 그대로 전달됩니다.

🔖 ② 구조화된 출처 목록

각 출처는 document_id, title, similarity, full_document 필드를 가지며 그래프 엔티티·관계 참조도 포함됩니다. 웹 질의응답은 답변과 별도로 이 출처를 전달해 근거 문서를 표시합니다.

이 조립 로직은 MCP search_context 도구와 웹 질의응답이 동일하게 사용합니다.
