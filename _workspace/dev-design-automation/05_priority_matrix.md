# 05. 우선순위 매트릭스 (영향력 × 구현 난이도)

> 영향력 = 기대효과 × 적용 빈도. 난이도 = 작업량 × 의존성/리스크(그래프 품질, 인프라).
> 약어: D=개발 시나리오(02), S=설계 시나리오(03), E=선행 enabler(04).

## 5.1 2×2 사분면

```
영향력 高
   │
   │  ┌─────────────────────────────┐   ┌─────────────────────────────┐
   │  │  ★ QUICK WINS (먼저)          │   │  ◆ BIG BETS (계획적 투자)     │
   │  │  D-1 사내지식 Q&A/IDE          │   │  D-2 코드 리뷰 자동화          │
   │  │  S-1 의존성/레이어 위반        │   │  S-2 영향도 분석(정밀)         │
   │  │  D-9 시맨틱 코드검색           │   │  E3 호출(call) 그래프         │
   │  │  S-7 설계 컨텍스트 조립        │   │  D-4 / S-10 부채·안티패턴      │
   │  │  (+ 전제) E1 CLI · E2 인덱싱   │   │                             │
   │  └─────────────────────────────┘   └─────────────────────────────┘
   │  ┌─────────────────────────────┐   ┌─────────────────────────────┐
   │  │  ○ FILL-INS (여력 시)         │   │  △ DEFER/AVOID (보류)        │
   │  │  D-8 커밋/PR 메시지           │   │  S-4 아키文서 자동업데이트(E4) │
   │  │  S-9 통합품질 모니터          │   │  S-5 설계 drift 검증(품질⚠)   │
   │  │  S-3 다이어그램 export        │   │  D-3 설계문서 자동생성(검수비용)│
   │  │  S-8 ADR 보조                │   │                             │
   │  └─────────────────────────────┘   └─────────────────────────────┘
   │
영향력 低 ─────────── 난이도 低 ──────────────────── 난이도 高 ───────►
```

## 5.3 권고 로드맵 (단계적)

**Phase A — 기반 (P0, ~1주):** E1(CLI) + E2(코퍼스 인덱싱) + **D-1**(사내지식 Q&A/IDE 연동).
→ 산출: Claude Code가 사내 코드·문서를 질의하는 동작 환경. 즉시 일상 생산성 + 이후 모든 시나리오의 토대.

**Phase B — 결정론 설계 분석 (P1, ~1~2주):** S-1(의존성/레이어) + D-9(코드검색) + S-7(설계 컨텍스트) + E3(call 그래프) 착수.
→ 산출: 고신뢰 의존/구조 분석 리포트. 품질 리스크 없는 빠른 가치.

**Phase C — 고효과 자동화 (P2, ~2~4주):** D-2(코드 리뷰) + S-2(정밀 영향도) + D-4/S-10(부채·안티패턴) + D-5(triage). E3 완료를 전제로 정밀도 확보.
→ 산출: PR 파이프라인에 붙는 리뷰/영향분석 자동화.

**Phase D — 보강/보류 (P3~P4):** Fill-in 항목은 여력 시. S-4/S-5는 그래프 품질·diff 인프라 성숙 후 재평가(현재 보류).

## 5.4 한 줄 결론

> **building block은 이미 충분히 성숙**하다. 자동화를 막는 것은 코드가 아니라 **(1) 코퍼스 데이터(E2), (2) 자동화 트리거 표준 CLI(E1), (3) 호출 그래프 한 단계(E3)** 다. 이 셋을 깔면 D-1·S-1을 필두로 한 Quick Win이 수일 내 가시화되고, 코드 리뷰·영향도 분석 같은 고효과 자동화로 자연스럽게 확장된다.
